# 自适应市场风格交易模式

## 1. 模式目标

这套模式不预设固定持仓周期，而是先识别市场环境和主导风格，再决定交易方法：

- 趋势风格：以趋势中军、弹性先锋为主，通常持有 3～10 个交易日。
- 投机连板风格：以情绪龙头、弹性先锋为主，通常按隔日节奏处理。
- 混合或轮动风格：降低仓位和频率，分别管理不同类型的交易。

完整决策链为：

> 风险硬门槛 → 赚钱/亏钱效应 → 风格识别 → 周期定位 → 板块迁移 → 主线确认 → 核心分类 → 次日确认 → 仓位、持有与退出

任何个股交易必须依次通过四道门：

> 市场允许交易 → 风格匹配 → 主线成立 → 核心出现买点

## 2. 使用边界

- 可交易池：沪深主板和创业板。
- 默认剔除：ST、退市整理、上市不足 60 日、停牌及长期低流动性个股。
- 科创板、北交所：只用于观察市场风格和赚钱效应，不进入候选交易池。
- 数据来源：通达信 MCP。
- 判断节奏：收盘后完成初判，次日盘中确认或推翻预判。
- 阈值原则：优先采用近 120 个交易日历史分位；文中的固定分值只是第一版起点，需要用后续 1 日、3 日和 5 日收益校准。

## 3. 每日判断流程

### 3.1 收盘后

1. 计算赚钱效应 `M` 和亏钱效应 `L`。
2. 计算趋势风格 `T` 和投机风格 `S`。
3. 根据指标当前值和 3 日变化方向判断周期。
4. 识别板块内切、横切、外切和高低切。
5. 计算各板块主线强度 `C`。
6. 在主线内识别情绪龙头、趋势中军和弹性先锋。
7. 输出次日观察条件、允许模式、仓位上限和失效条件。

### 3.2 次日盘中

- 检查昨日强势股开盘溢价及开盘后 30 分钟修复情况。
- 检查主线板块强度、成交占比和核心承接。
- 检查高标是否出现核按钮及负反馈扩散。
- 检查大盘黄白线劈叉是否与大小市值收益、成交方向一致。
- 盘中信号只负责确认或降低仓位；普通风格切换至少等待收盘确认，避免被脉冲行情误导。

## 4. 赚钱效应与亏钱效应

赚钱效应和亏钱效应分别评分，不能简单相减。除全市场有效交易池外，额外维护昨日涨停及强势股池、连板股池、趋势核心池和高成交额核心池。

### 4.1 赚钱效应 `M`（100 分）

| 维度 | 权重 | 主要指标 |
| --- | ---: | --- |
| 市场广度 | 20 | 上涨家数占比、涨跌幅中位数、涨幅超过 5% 的数量 |
| 强势股次日溢价 | 25 | 昨日涨停股、放量突破股次日开盘及收盘收益中位数 |
| 持续性 | 20 | 连板晋级率、创 20 日新高后的延续率、趋势股沿 5/10 日线运行比例 |
| 高度与空间 | 15 | 连板高度、近 5 日累计涨幅领先股的后续表现 |
| 容量赚钱效应 | 20 | 成交额前 100 名的涨跌幅中位数、创新高数量和趋势延续率 |

### 4.2 亏钱效应 `L`（100 分）

| 维度 | 权重 | 主要指标 |
| --- | ---: | --- |
| 极端亏损 | 25 | 跌停数、跌幅超过 7% 的数量、天地板和单日大幅回撤数量 |
| 强势股负反馈 | 25 | 昨日涨停低开/收跌比例、昨日炸板股次日继续下跌幅度 |
| 高位股风险 | 25 | 高位核心跌停、连续大阴线、断板后无法修复的比例 |
| 趋势破坏 | 15 | 趋势核心跌破 10 日线、放量破位及板块核心批量补跌 |
| 亏钱扩散度 | 10 | 负反馈是否从局部扩散至连板、趋势和容量股 |

### 4.3 四象限解释

| 状态 | `M` | `L` | 交易含义 |
| --- | --- | --- | --- |
| 顺风进攻 | 高 | 低 | 主升或强修复，可以积极参与 |
| 高波动分歧 | 高 | 高 | 机会与风险并存，只做核心 |
| 混沌轮动 | 低 | 低 | 缺少持续性，降低频率 |
| 退潮防守 | 低 | 高 | 控仓或空仓，等待负反馈释放 |

输出不能只有数字，还要指出贡献来源，例如：`M=72，容量趋势贡献最大；L=61，负反馈集中在连板高位股。`

## 5. 市场风格识别

趋势与投机分别评分，不强制二选一。风格判断的核心不是外观，而是对应交易方法是否产生实际收益。

### 5.1 趋势风格 `T`（100 分）

| 维度 | 权重 | 主要指标 |
| --- | ---: | --- |
| 趋势策略实际收益 | 30 | 放量突破、创 20 日新高、沿均线上涨样本的后 3/5 日收益和胜率 |
| 趋势结构完整度 | 20 | 多头排列、回踩修复、突破后不跌回平台的比例 |
| 容量核心表现 | 20 | 成交额前 100 名上涨及创新高比例、核心持续放量能力 |
| 持续时间 | 15 | 领先股是否连续推进，而非单日脉冲后回落 |
| 板块趋势共振 | 15 | 中军、弹性股、补涨股和板块指数是否共同维持趋势 |

### 5.2 投机风格 `S`（100 分）

| 维度 | 权重 | 主要指标 |
| --- | ---: | --- |
| 隔日投机实际收益 | 30 | 昨日涨停股次日开盘/收盘溢价及打板样本胜率 |
| 连板晋级结构 | 25 | 一进二、二进三及高位晋级率 |
| 高度与梯队 | 15 | 最高板空间和首板、二板、中高位梯队完整度 |
| 小票相对强度 | 15 | 小市值活跃股相对容量股的超额收益、成交集中度和涨停贡献 |
| 情绪修复能力 | 15 | 弱转强、断板反包、炸板股次日修复能力 |

### 5.3 初始判定

- `T≥65` 且 `T-S≥15`：明确趋势风格。
- `S≥65` 且 `S-T≥15`：明确投机风格。
- `T≥65` 且 `S≥65`：趋势与投机共振，继续判断主线载体。
- `T`、`S` 均在 45～65：混合轮动，没有稳定风格。
- `T`、`S` 均低于 45：两种方法都缺少赚钱效应，以防守为主。

### 5.4 大盘黄白线

黄白线只作为盘中辅助指标，建议占风格评分的 5%～10%，不能独立决定市场风格：

- 黄线持续强于白线：中小票相对占优，可能支持题材投机。
- 白线持续强于黄线：权重和容量股相对占优，可能支持趋势风格。
- 白线上涨、黄线下跌：可能只是权重护盘，多数股票赚钱效应较差。
- 黄线上涨、白线下跌：可能是指数偏弱下的结构性题材行情。

必须同时验证黄白线差值持续时间、大小市值分组收益差、成交额集中方向和涨停股隔日溢价。

## 6. 周期状态机

周期按以下顺序演化：

> 退潮 → 冰点衰竭 → 试错修复 → 主升 → 高位震荡 → 再次退潮

轮动是市场结构标签，不是独立周期阶段；试错期和高位震荡期都可能伴随轮动。

核心观测量：

- `M`：赚钱效应及 3 日变化方向。
- `L`：亏钱效应及 3 日变化方向。
- `C`：主线集中度。
- `P`：连板晋级、趋势延续和强势股次日溢价所代表的持续性。
- `R`：分歧、炸板和破位后的修复能力。

| 阶段 | 锚点 | 操作含义 |
| --- | --- | --- |
| 退潮 | `M` 下降、`L` 快速上升、核心补跌、负反馈扩散 | 控仓或空仓 |
| 冰点衰竭 | `L` 仍高但不再恶化，成交萎缩，高位风险释放 | 观察，不因冰点直接抄底 |
| 试错修复 | `L` 下降、`R` 上升，新方向出现但持续性未确认 | 小仓验证 |
| 主升 | `M`、`C`、`P` 同步上升，`L` 低，分歧后快速修复 | 聚焦主线核心 |
| 高位震荡 | `M` 仍高但边际下降，`L` 上升，核心波动和掉队增加 | 去弱留强、降低仓位 |

普通阶段切换需要连续两个收盘信号确认；核心跌停、负反馈快速扩散等极端风险可以当天触发防守。

投机周期重点观察连板高度、晋级率、隔日溢价和核按钮扩散；趋势周期重点观察板块趋势、容量核心、突破样本后续收益和破位扩散。两套锚点不能混用。

## 7. 板块切换与资金迁移

| 类型 | 资金路径 | 通常含义 |
| --- | --- | --- |
| 板块内切 | 主线高位 → 主线低位 | 主线内部补涨或高低切 |
| 板块横切 | 强板块 A → 同级板块 B | 缺少绝对主线，偏轮动 |
| 板块外切 | 原主线 → 全新题材 | 新周期试错或主线切换 |
| 高低切 | 高位股 → 低位股 | 风险偏好下降或健康扩散 |

每日记录板块 1/3/5 日相对强度、成交额占比、涨停/连板/创新高数量、核心及后排次日溢价、前五强板块连续上榜率。

### 7.1 健康的板块内高低切

- 原核心高位震荡，但没有连续严重负反馈。
- 低位补涨主动走强。
- 板块指数趋势未破坏。
- 板块成交额保持或增加。
- 容量中军仍有承接。

如果核心跌停、板块指数破位，只剩低位补涨，则属于撤退过程中的残余轮动，不是健康扩散。

### 7.2 板块横切

近 3 日领涨板块频繁变化、前五强重复率低、单个板块通常只强 1 天、次日溢价弱且市场没有明显增量成交时，定义为存量资金横切。此时不把当日最强板块直接定义为主线。

### 7.3 板块外切和主线切换

确认主线切换需要同时满足：

1. 原主线核心和板块强度持续下降。
2. 新方向连续两日保持强度。
3. 新方向成交额占比持续提高。
4. 新方向形成先锋、容量中军和完整梯队。
5. 分歧后仍有资金回流。
6. 新方向次日溢价高于原主线。

只有旧主线退位，没有新方向承接，属于退潮；新方向上涨但旧主线仍强，可能是双主线或支线轮动。

## 8. 主线板块确认

主线不是当日涨幅第一，而是持续承载市场赚钱效应和成交资金的方向。

### 8.1 主线强度 `C`（100 分）

| 维度 | 权重 | 主要指标 |
| --- | ---: | --- |
| 持续性 | 20 | 1/3/5 日强度排名和连续上榜情况 |
| 资金承载 | 20 | 板块成交额、全市场成交占比及增量 |
| 赚钱效应 | 20 | 涨停、创新高、趋势上涨和次日溢价 |
| 梯队完整 | 15 | 先锋、情绪龙头、趋势中军和低位补涨是否齐全 |
| 相对强度 | 15 | 板块指数相对大盘的超额收益和抗跌性 |
| 分歧修复 | 10 | 分歧后能否回流，核心能否再次带动板块 |

初始规则：

- `C≥70` 且连续两个交易日成立：主线候选。
- `C≥75` 且明显领先第二名：确认主线。
- `60≤C<70`：支线或轮动方向。
- 单日爆发但缺少持续性：热点，不定义为主线。

以下情况触发主线硬排除：只有一只股票上涨、上涨由低流动性个股贡献、首日爆发后无次日溢价、缺少容量中军、核心上涨但板块指数持续走弱，或板块负反馈扩散后无法修复。

## 9. 核心龙头分类

### 9.1 情绪龙头

- 主动打开市场高度，而不是被动成为最高板。
- 在板块启动或关键转折时率先走强。
- 分歧时有承接，修复时率先反包或回封。
- 强弱变化能够影响板块内其他个股。
- 具有高度、辨识度和隔日流动性。

### 9.2 趋势中军

- 板块内成交额和资金容量领先。
- 与板块指数高度相关，并对板块强弱有实际影响。
- 维持稳定趋势，回踩后有大资金承接。
- 分歧时抗跌，回流时能够持续放量。
- 可以容纳较大仓位，不依赖瞬间封板。

### 9.3 弹性先锋

- 板块启动时涨速和超额收益领先。
- 对消息、情绪和资金回流最敏感。
- 上涨弹性显著高于中军。
- 率先突破或率先修复。
- 具备足够流动性，不能只是后排跟涨小票。

三类核心分别按主动性 25 分、板块影响力 25 分、资金认可度 20 分、分歧承接与修复 20 分、稀缺性和辨识度 10 分计算。最重要的验证是个股与板块的领先—跟随关系，而不是静态比较涨幅、连板数或成交额。

## 10. 仓位、持有和退出

### 10.1 周期仓位上限

| 周期 | 仓位上限 | 核心任务 |
| --- | ---: | --- |
| 退潮 | 0%～20% | 防守，停止模式外交易 |
| 冰点衰竭 | 0%～20% | 观察修复，不因超跌直接买入 |
| 试错修复 | 20%～40% | 小仓验证新方向 |
| 主升 | 50%～80% | 聚焦主线核心 |
| 高位震荡 | 30%～50% | 去弱留强，避免后排 |
| 快速轮动 | 20%～30% | 降低频率，不追当日高潮 |

仓位上限不是必须仓位；亏钱效应硬门槛可以进一步降低上限。

### 10.2 趋势交易

- 标的：趋势中军、弹性先锋。
- 买点：放量突破、首次有效回踩、板块分歧后的率先修复。
- 持有：通常 3～10 个交易日。
- 退出：个股趋势破坏、板块强度持续下降或核心地位被替代。

### 10.3 投机交易

- 标的：情绪龙头、最强弹性先锋。
- 买点：首次分歧转一致、弱转强、关键晋级。
- 持有：通常按隔日反馈处理。
- 退出：次日反馈低于预期、梯队瓦解或核心失去主动性。

混合共振环境下，趋势仓位和投机仓位必须分别设置退出条件，不能用趋势逻辑为投机股找理由。

### 10.4 风险硬门槛

以下任一条件出现，停止对应模式的新开仓：

- `L≥75` 且继续上升。
- 高位核心负反馈向中低位扩散。
- 昨日强势股整体无溢价。
- 主线核心与板块指数同步破位。
- 新方向只有单日爆发，没有次日承接。
- 市场成交下降，同时板块轮动加快。

### 10.5 单笔交易检查

1. 当前周期允许多少仓位？
2. 当前赚钱的是哪种模式？
3. 标的是否属于确认主线？
4. 它是哪一类核心？
5. 当前买点属于突破、回踩还是分歧修复？
6. 哪个客观条件出现，能够证明判断错误？

失效条件必须在买入前确定。

```text
市场周期：试错修复
赚钱效应 M：67，连续 2 日回升
亏钱效应 L：42，继续下降
风格：趋势 T=73，投机 S=48
市场结构：主线内部高低切
主线：板块 A，强度 C=76
支线：板块 B，强度 C=63

情绪龙头：个股 X
趋势中军：个股 Y
弹性先锋：个股 Z

次日观察：
1. 板块 A 成交占比能否继续提高
2. 个股 Y 回踩是否获得承接
3. 昨日强势股是否保持正溢价

仓位上限：40%
允许模式：趋势首次回踩
禁止模式：追逐轮动板块高潮
失效条件：板块 A 跌出 3 日强度前五，核心同步破位
```

## 12. 后续数据验证

下一阶段通过通达信 MCP 完成以下工作：

1. 明确全部指标的字段、样本池、复权和异常值处理口径。
2. 用滚动历史分位替代未经验证的固定阈值。
3. 检验各评分对后续 1 日、3 日和 5 日收益及回撤的解释力。
4. 检验阶段切换信号的稳定性，避免状态频繁跳变。
5. 分别验证趋势样本和投机样本，防止混用指标。
6. 保留每日原始指标、评分、结论和实际后续收益，持续复盘修正。

## 13. 核心原则

> 先判断哪里能够赚钱，再判断用什么方式赚钱；随后判断处于什么阶段、资金正在往哪里迁移，最后只交易主线中具有真实影响力的核心。

该框架用于形成一致、可验证的交易决策流程，不代表收益保证。任何评分和阈值都必须经过历史样本及持续实盘记录验证。

## 14. Web 决策站点

项目包含 React 决策驾驶舱和 FastAPI 数据服务：

- `frontend/`：登录、盘中驾驶舱、市场图谱、主线、核心、舆情和历史工作区。
- `backend/`：认证、SQLite 快照、通达信 MCP 客户端、规则引擎和 WebSocket。
- `docker-compose.yml`：Nginx 通过 19098 提供站点，后端只在容器网络使用 19099。

本地启动前复制 `.env.example` 为 `.env`，至少修改站点密钥和管理员密码。生产环境的通达信 token 只保存在服务器 `.env`，不得提交 Git。

```bash
docker compose up -d --build
```

健康检查：`GET /api/health`。当 MCP 不可用时，系统保留最近一次成功快照并明确显示数据延迟，不会把空响应显示成零。
