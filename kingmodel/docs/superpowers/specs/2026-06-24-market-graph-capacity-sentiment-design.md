# 市场图谱、容量核心与舆情复盘接入设计

## 目标

修复三个决策缺口：

1. 市场图谱不能与主线板块重复，必须展示资金迁移和带动关系。
2. 趋势/容量行情下必须单列容量核心，并允许其进入明日计划候选池。
3. 隔夜舆情和盘后复盘必须进入计划评分与验证条件，但不能绕过盘面确认。

## 数据结构

- `market_graph`：节点和连线。节点包括市场状态、容量反馈、负反馈、主线、板块联动、龙头、容量核心、计划标的。
- `capacity_cores`：从同日全市场日线按成交额、涨跌幅、主线匹配、板块联动、可交易性筛选。
- `event_signals`：把隔夜舆情、盘后复盘、模型收益回填转换成可审计信号。

## 计划评分影响

容量核心会转换为 `趋势容量核心` 候选，参与 `planned_targets` 排名。事件信号只作为加分和验证条件：

- 主线匹配、联动匹配、热度合理时加分。
- 舆情拥挤但无板块联动时扣分。
- 最终买入仍必须满足计划里的分时承接、板块扩散和禁买条件。

## UI

- 市场图谱页：四列关系图，展示负反馈 → 市场/容量 → 主线/联动 → 核心/计划。
- 主线板块页：只做板块诊断，不再复用市场图谱。
- 核心标的页：先展示容量核心，再展示全部候选池。
- 驾驶舱：新增容量核心；隔夜预期改为舆情/盘后复盘信号。

## Token 约束

本次改动不增加前端自动刷新频率，不引入通达信循环调用。容量核心优先使用已有同日全市场日线，舆情/复盘先使用本地可审计字段；未接入真实外部源时不伪造来源。
