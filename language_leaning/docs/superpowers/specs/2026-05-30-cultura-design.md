# Cultura — 故事驱动的语言 + 文化学习 App 设计

> 日期: 2026-05-30
> 状态: 已批准

---

## 一、产品定位

面向 20-45 岁语言爱好者和文化探索者，通过 AI 生成的历史文化故事驱动语言学习。不同于 Duolingo 的碎片化练习模式，Cultura 让用户在阅读真实历史文化故事的过程中自然习得语言，同时深入了解目标语言背后的文化脉络。

- **首发语言**：中文 + 英语（双向互学）
- **平台**：Android（Flutter 跨平台，后续可扩展 iOS）
- **应用名**：Cultura

## 二、目标用户

- 20-45 岁语言爱好者 / 文化探索者
- 对目标语言所在国家的历史文化有浓厚兴趣
- 有一定自学能力，追求深度而非速成
- 愿意为高质量内容付费

## 三、核心功能模块

### 1. 故事大厅 (Story Hall)
- 按时代、地区、语言难度筛选和浏览历史故事
- 每个故事标注：历史时期、文化标签、语言等级（CEFR A1-C2）
- 推荐系统：根据用户学习进度和兴趣推荐故事

### 2. 沉浸式阅读 (Immersive Reader)
- 双语对照阅读模式（原文 + 母语翻译）
- 点击任意词汇查看：释义、发音（AI TTS）、用法示例、词源
- 生词自动收录到个人生词本
- 段落级 AI 朗读

### 3. 故事练习 (Story Exercises)
- AI 根据故事内容动态生成练习题：
  - 词汇填空（故事语境中的词汇）
  - 语法练习（故事中出现的语法点）
  - 听力练习（AI 朗读故事片段）
  - 阅读理解（基于故事内容的问答）
- 错题自动收录，间隔重复复习

### 4. 文化地图 (Culture Map)
- 时间线 + 地图可视化展示已学习的文化知识点
- 按历史时期和地理区域组织
- 每个节点关联已读故事和相关词汇

### 5. AI 角色扮演对话 (AI Roleplay)
- 与故事中的历史人物进行对话练习
- AI 根据故事背景扮演角色，用户用目标语言交流
- 实时语法纠错和表达建议

### 6. 个人中心 (Profile)
- 学习进度追踪（已读故事数、掌握词汇量、连续学习天数）
- 个人生词本（含间隔重复复习）
- 成就徽章系统
- 模型偏好设置（选择 AI 模型）

## 四、学习流程

```
1. 选择故事（按时代/地区/语言难度筛选）
2. 沉浸式阅读（双语对照 + 点击查词 + AI 发音）
3. 文化探索（历史背景注释、文化延伸、地图标记）
4. AI 生成练习（词汇/语法/听力/阅读理解）
5. AI 角色扮演对话（与故事人物互动巩固）
```

## 五、技术架构

### 客户端
- **框架**：Flutter (Dart)
- **本地存储**：SQLite / Hive（离线阅读、生词本、学习数据）
- **状态管理**：Riverpod 或 Bloc

### 后端
- **API 框架**：Python FastAPI（或 Node.js）
- **数据库**：PostgreSQL（用户数据、学习记录）
- **缓存**：Redis（API 响应缓存、会话管理）
- **对象存储**：S3 兼容（音频、图片等媒体资源）

### AI 模型网关 (Model Gateway)
- 统一模型路由层，抽象各模型 API 差异
- 支持模型列表：
  - **Claude (Anthropic)** — 高质量，高成本
  - **DeepSeek** — 性价比高
  - **Qwen (通义千问)** — 中文强，低成本
  - **Kimi (月之暗面)** — 长文本，中文好
  - **MiniMax** — 对话强，角色扮演
  - 可扩展接入更多模型
- 用户可在设置中自由切换默认模型
- 简单任务（练习生成）自动路由到低成本模型
- 复杂任务（故事生成）使用高端模型

### 内容生成策略
- 全部内容由 AI 实时生成（Claude API 等）
- 故事生成：基于真实历史事件，AI 生成叙事文本 + 语言学习标注
- 练习生成：基于故事内容动态生成
- 对话：AI 角色扮演

## 六、商业模式

### 三级订阅体系

| 功能 | 免费版 | 包月 Pro (¥98/月) | 包年 Pro (¥998/年) |
|------|--------|-------------------|---------------------|
| 故事阅读 | 3 个/天 | 无限 | 无限 |
| AI 对话 | 10 条/天 | 200 条/天 | 500 条/天 |
| 练习功能 | 基础 | 全部 | 全部 |
| 模型切换 | 默认模型 | 可切换 | 可切换 + 优先高端 |
| 离线下载 | 无 | 有 | 有 |
| 专属内容 | 无 | 无 | 有 |

包年 ≈ ¥83/月，比包月省 ¥182/年。

### API 成本控制策略
1. **响应缓存** — 相同故事/练习请求直接返回缓存结果
2. **分级模型** — 故事生成用高端模型，练习生成用低成本模型
3. **批量预生成** — 非高峰期批量生成热门内容并缓存
4. **配额硬限制** — 按订阅等级严格限制每日 API 调用次数
5. **智能路由** — 简单任务自动分配到低成本模型

## 七、关键差异化

| Cultura | Duolingo / Babbel |
|---------|-------------------|
| 故事驱动学习 | 碎片化练习 |
| 深度历史文化内容 | 纯语言技能 |
| AI 生成无限内容 | 固定课程 |
| 角色扮演对话练习 | 机械重复 |
| 文化地图可视化 | 无文化维度 |
| 多模型自由选择 | 无 AI 对话 |

## 八、挑战与风险

1. **AI 内容质量** — 需持续优化 prompt 工程，确保故事的历史准确性和语言教学的科学性
2. **API 成本** — 通过分级模型、缓存、配额控制来管理
3. **目标用户群体** — 相对小众，需精准营销
4. **多模型兼容性** — 需实现统一的模型抽象层（Provider Pattern），处理各模型 API 差异和限流策略
