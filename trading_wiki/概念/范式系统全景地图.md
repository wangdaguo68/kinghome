# 范式系统全景地图

## 一句话定义

范式系统研究知识、技术、组织和文明如何从旧框架切换到新框架。它帮助你理解“为什么有些变化不是线性改良，而是重新定义问题、规则和价值”。

## 使用方式

范式系统只作为顶层观察框架，不在菜单里继续拆很多子入口。后续具体笔记可以作为普通知识页存在，但导航只保留本页，避免把系统入口做成一堆概念菜单。

## 范式系统的核心问题

| 问题 | 为什么重要 |
| --- | --- |
| 什么是旧范式无法解释的异常？ | 新范式通常从异常积累开始 |
| 新技术是否只是工具，还是改变系统结构？ | 真正的大机会来自结构重构 |
| 扩散需要哪些基础设施？ | 技术本身不等于产业化 |
| 谁会被新范式赋能，谁会被淘汰？ | 判断权力、利润和组织变化 |
| 未来有几种可能路径？ | 避免单线预测 |
| 我能主动定义什么方向？ | 从被动适应转向战略设计 |

## 与投资和个人成长的关系

| 场景 | 范式视角 |
| --- | --- |
| AI | 不是只看模型参数，而是看知识生产、软件形态、组织流程和产业链重构 |
| 机器人 | 不是只看硬件，而是看劳动力、制造系统和服务场景重构 |
| 医药 | 不是只看单药，而是看平台技术、支付系统和监管范式 |
| 品牌 | 不是只看营销，而是看文化符号和消费身份变迁 |
| 个人成长 | 不是只学技巧，而是更新自己的问题定义和行动系统 |

## 常见误区

| 误区 | 更准确理解 |
| --- | --- |
| 新技术一定马上替代旧系统 | 扩散需要成本下降、基础设施、制度和用户习惯 |
| 范式转移就是热点词 | 真正范式转移会改变问题、方法、组织和价值分配 |
| 未来学就是预测 | 未来学更重视情景、驱动因素和不确定性 |
| 系统科学就是画复杂图 | 系统分析要识别反馈、边界、延迟和杠杆点 |
| 战略就是定目标 | 战略是选择方向、取舍路径和塑造系统 |
