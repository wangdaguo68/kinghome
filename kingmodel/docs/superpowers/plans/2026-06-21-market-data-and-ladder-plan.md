# 市场数据校验与梯队重构实施计划

1. 扩展配置，加入 Tushare 备用源 Token 和接口地址。
2. 新增通达信结果语义校验模块，覆盖广度、涨跌停、连板、板块和成交额排序。
3. 新增 Tushare 日线备用采集器，计算全 A 广度、涨跌幅中位数、涨跌停、炸板和成交额前 100 反馈。
4. 重构采集器：通达信逐项主采、语义失败后逐项降级、无效结果不覆盖最近可信值。
5. 使用通达信 `limit_up_analysis` 结构化接口补齐连板标的涨停主题、原因揭秘、因素类型和置信度。
6. 扩充核心梯队为连板情绪、趋势容量、创业板 20cm 三组，并生成完整连板梯队数据结构。
7. 更新前端数据类型和驾驶舱：删除 HTTP 提示、正负反馈并排、实时容量反馈、分组核心梯队、全宽连板梯队。
8. 增加语义校验、备用源计算、因素解析和交易候选过滤测试。
9. 运行后端测试、前端类型检查与生产构建，检查无静态错误风险文案残留。
10. 安全部署 Tushare Token，重建容器并验证主源、降级、最近可信快照和线上界面。
11. 提交并推送代码。
