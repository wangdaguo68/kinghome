# 范式理论与科学革命

## 概念定义

范式是一个共同体共享的问题定义、方法、标准、工具和典型案例。范式转移不是简单出现新观点，而是旧范式无法处理异常，新范式重新定义问题和解决方式。

## 范式更替过程

| 阶段 | 说明 | 例子 |
| --- | --- | --- |
| 常规科学 | 大家在旧框架内解决问题 | 沿用既有理论和工具 |
| 异常积累 | 越来越多现象难以解释 | 旧模型反复补丁 |
| 危机 | 共同体对旧框架信心下降 | 方法和标准开始争论 |
| 新范式出现 | 新框架解释异常并打开新问题 | 新理论、新工具、新语言 |
| 共同体转换 | 教育、制度、资源向新范式迁移 | 新教材、新公司、新基础设施 |

## 判断是否是范式转移

| 判断问题 | 如果答案是“是” |
| --- | --- |
| 是否改变问题定义？ | 不是只改答案，而是改问题 |
| 是否改变方法和工具？ | 需要新能力和新基础设施 |
| 是否改变评价标准？ | 旧指标可能失效 |
| 是否改变组织和权力结构？ | 新玩家获得优势 |
| 是否产生新语言？ | 出现新概念、新分类、新叙事 |

## 投资中的用法

不要把每个热点都叫范式转移。真正的大机会通常伴随：成本曲线变化、基础设施成熟、用户行为改变、监管或制度适配、利润池迁移。

## 常见误区

| 误区 | 更准确理解 |
| --- | --- |
| 范式转移等于新技术发布 | 技术发布只是起点，系统采用才关键 |
| 旧范式会理性让位 | 旧体系有利益、教育和组织惯性 |
| 新范式一开始就更完善 | 新范式早期常粗糙，但解释力和潜力更大 |
| 范式理论可以证明任何趋势 | 它是分析框架，不是万能标签 |
