# 机器学习进度面板设计

## 目标

在决策驾驶舱中增加一个可直接判断机器学习是否工作的状态面板。面板只展示状态，不改变现有交易计划、影子 Top3、模型训练和数据采集逻辑。

## 数据来源

复用现有 `/api/dashboard` 字段：

- `feature_store_status`：特征交易日、结果标签交易日、最新样本日期。
- `ml_system`：当前阶段、训练门槛、模型数量、最近训练记录、模块状态。
- `ml_shadow`：当前影子 Top3。
- `ml_review`：1/3/5/10 日结果标签统计和明细。

不新增通达信 MCP 调用，不新增行情接口调用。

## 界面结构

面板放在正式“明日计划标的”和“模型影子 Top3”之间，避免用户把影子 Top3 误认为实盘模型推荐。

展示内容：

1. 当前阶段：规则影子积累、影子学习、模型辅助、实盘可参考。
2. 模型参与状态：是否有 champion 模型、是否参与正式计划排序。
3. 样本进度：特征交易日、结果标签交易日、距离 20/60/120 天门槛的进度。
4. 结果回填：1/3/5/10 日样本数、胜率、平均收益。
5. 今日影子 Top3：明确标记为“只记样本，不参与实盘”。
6. 最近训练：训练版本、状态、样本数、完成时间。

## 风险控制

- 当 `champion_count = 0` 时，明确显示“未参与实盘排序”。
- 当样本不足时，显示距离首次训练还差多少个结果交易日。
- 面板不产生交易建议，只解释机器学习系统状态。
