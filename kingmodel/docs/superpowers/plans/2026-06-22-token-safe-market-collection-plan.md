# 低 Token 市场采集实施计划

1. 扩展配置和数据库，加入可信快照、每日收盘任务、免费涨停池、原因缓存和通达信调用审计。
2. 实现数据库原子预算控制器，保证每交易日最多 5 次且同股同日只调用一次。
3. 实现东方财富免费指数交易日与涨停池客户端，校验响应后持久化最近交易日数据。
4. 基于本地每日涨停池计算真实连续板和近 5 日板数，构建免费原因与收盘梯队。
5. 将通达信补查限制为收盘任务内部、免费原因不足的前 5 个核心；生产初始关闭该开关。
6. 将驾驶舱改为只读取最新可信发布版本；失败采集只记录诊断，不覆盖已发布数据。
7. 将手动刷新改为免费收盘缓存刷新，删除分钟级全量 MCP 采集循环。
8. 增加版本标签、刷新零 Token 提示、设置页预算和任务状态。
9. 增加并发预算、重启缓存、失败保留、手动刷新零 MCP 与真实连板测试。
10. 本地测试与生产构建通过后部署，保持 MCP 补查关闭，验证页面和调用计数，再提交推送。
