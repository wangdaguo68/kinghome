# 机器学习交易计划引擎实施计划

## 阶段 1：特征、标签与规则基线

1. 在数据库增加版本化市场、板块、个股特征快照，按交易日和特征版本唯一。
2. 保存每日候选池及次日、3 日、5 日、10 日结果，标签只能在对应未来交易日完成后回填。
3. 将赚钱效应、亏钱效应、趋势、投机、周期、主线和核心评分拆成独立纯函数。
4. 实现规则版动态交易周期、正期望、盈亏比和 Top3/空仓选择器。
5. 当历史样本不足时只展示规则得分，不伪造机器学习概率。
6. 增加时间边界、不可成交、收益标签、盈亏比和排序稳定性测试。
7. 生产部署为影子模式，继续使用当前计划，后台只积累特征和结果。

## 阶段 2：市场与板块模型

1. 回补至少 3 年市场和板块日频数据。
2. 建立按时间走步、带 10 日隔离区的数据集。
3. 训练并校准趋势、投机、周期和板块延续模型。
4. 输出概率、特征贡献、数据漂移和规则对照，不自动选股。
5. 通过多窗口走步回测后进入线上影子推理。

## 阶段 3：个股模型与 Top3

1. 分别建立连板、趋势容量和 20cm 样本。
2. 训练可成交性、收益、最大回撤和持有周期模型。
3. 使用校准胜率、盈亏比、正期望和每单位风险收益执行硬过滤。
4. 生成 0–100 综合得分，严格输出 Top1–Top3 或空仓。
5. 完成成本、涨跌停、停牌和不可成交约束下的走步回测。

## 阶段 4：持续学习

1. 自动回填计划后续 1、3、5、10 日结果。
2. 每周训练 Challenger，并与 Champion 多窗口比较。
3. 只有收益、回撤、概率校准和盈亏比同时达标才允许升级。
4. 增加模型注册、漂移监控、回滚和复盘界面。

## 发布门槛

- 任一阶段不得调用高频通达信 MCP。
- 训练与线上特征逐字段一致。
- 无未来数据泄漏。
- 机器学习模型先影子运行，未达到发布门槛前不改变交易计划。
- 每阶段独立提交、测试、部署和验收。
