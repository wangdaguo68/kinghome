# 运行指南

## 前置要求

在运行项目之前，您需要先安装以下软件：

### 1. 安装 Node.js

1. 访问 [Node.js 官网](https://nodejs.org/)
2. 下载并安装 LTS 版本（推荐 v18 或更高版本）
3. 安装完成后，重启命令行工具

### 2. 验证安装

打开新的命令行窗口，运行以下命令验证安装：

```bash
node --version
npm --version
```

如果显示版本号，说明安装成功。

### 3. 初始化数据库

确保 MySQL 服务已启动，然后执行：

```bash
mysql -u root -pking665206 < database/init.sql
```

或者在 MySQL 客户端中执行 `database/init.sql` 文件的内容。

### 4. 配置环境变量

确保项目根目录下有 `.env.local` 文件，内容如下：

```
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=king665206
DB_NAME=kingblog
```

## 运行项目

### 步骤 1: 安装依赖

在项目根目录下运行：

```bash
npm install
```

### 步骤 2: 启动开发服务器

```bash
npm run dev
```

### 步骤 3: 访问应用

打开浏览器访问：http://localhost:3000

## 常见问题

### 问题 1: npm 命令未找到

**解决方案**: 确保已安装 Node.js，并重启命令行工具。如果问题仍然存在，检查系统环境变量 PATH 是否包含 Node.js 的安装路径。

### 问题 2: 数据库连接失败

**解决方案**: 
- 检查 MySQL 服务是否已启动
- 确认数据库配置信息是否正确（`.env.local` 文件）
- 确认数据库 `kingblog` 是否已创建

### 问题 3: 端口 3000 已被占用

**解决方案**: 
- 关闭占用 3000 端口的程序
- 或者修改 `package.json` 中的 dev 脚本，添加端口参数：
  ```json
  "dev": "next dev -p 3001"
  ```

## 项目命令

- `npm run dev` - 启动开发服务器
- `npm run build` - 构建生产版本
- `npm start` - 启动生产服务器
- `npm run lint` - 运行代码检查
