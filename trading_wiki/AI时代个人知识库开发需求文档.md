# AI时代个人知识库系统开发需求文档

## 1. 项目名称

**AI时代个人知识库大全 / 个人认知操作系统**

可选产品名：

- 知识宇宙
- 认知操作系统
- AI时代个人知识库
- 九大系统知识库
- Personal Knowledge OS

---

## 2. 项目背景

在AI时代，知识获取越来越容易，但真正稀缺的是：

- 能否建立自己的知识体系；
- 能否把碎片信息沉淀成结构化认知；
- 能否用知识解释现实世界；
- 能否从知识中发现产品、产业、投资和个人成长机会；
- 能否形成自己的审美、判断力和范式思维。

本项目不是普通的笔记软件，也不是资料收藏夹，而是一个面向AI时代的 **个人认知操作系统**。

它的核心目标是帮助用户围绕九大系统沉淀长期知识：

1. 认知系统：人如何思考
2. 智能系统：AI如何工作
3. 产业系统：技术如何变成产业
4. 财富系统：价值如何流动
5. 生命系统：生命如何运转
6. 物理系统：现实世界如何运转
7. 美学系统：什么是美
8. 意义系统：什么值得被记住
9. 范式系统：新旧世界如何更替

---

## 3. 产品定位

### 3.1 一句话定位

一个帮助用户构建AI时代个人知识体系、沉淀认知模型、连接产业机会和现实决策的智能知识库系统。

### 3.2 产品不是

- 不是单纯的笔记工具；
- 不是简单的文章收藏工具；
- 不是百科全书；
- 不是普通AI聊天工具。

### 3.3 产品是

- 个人知识库；
- 认知模型库；
- 产业研究库；
- AI辅助学习系统；
- 个人判断力训练系统；
- 长期主义知识沉淀系统。

## 4. 核心设计思想

每个知识点都必须从“信息”升级为“认知资产”。

系统中的每条知识卡片都应尽量包含以下结构：

```md
# 概念名称

## 所属系统
认知系统 / 智能系统 / 产业系统 / 财富系统 / 生命系统 / 物理系统 / 美学系统 / 意义系统 / 范式系统

## 一句话解释
用最通俗的话解释这个概念。

## 核心原理
它为什么成立？底层逻辑是什么？

## 关键模型
可以用什么框架理解它？

## 典型案例
现实世界中哪里发生过？

## 相关产业
它对应哪些行业、公司、产品、技术、趋势？

## 我的启发
这对我有什么启发？

## 可迁移场景
它还能用在哪些地方？

## 风险和误区
使用这个概念时容易犯什么错？

## 关联知识
它和哪些知识点有关？
```

核心原则：

> 不只收藏知识，而是让知识可以被调用、被组合、被迁移、被用于判断现实。

## 6.4 AI辅助生成知识卡片


AI需要自动生成：

- 所属系统
- 一句话解释
- 核心原理
- 关键模型
- 典型案例
- 相关产业
- 我的启发
- 可迁移场景
- 风险和误区
- 关联概念
- 推荐标签

### Prompt模板

```md
请把【{概念名称}】整理进我的个人知识库。

要求：
1. 判断它属于九大系统中的哪一个或哪几个：
认知系统、智能系统、产业系统、财富系统、生命系统、物理系统、美学系统、意义系统、范式系统。

2. 用通俗语言解释它。
3. 说明它的底层原理。
4. 提炼3-5个可复用模型。
5. 给出3个现实案例。
6. 如果和AI、A股、产业链、产品设计、个人成长有关，请单独列出来。
7. 输出我的启发、可迁移场景、风险和误区、关联概念。
8. 使用Markdown格式输出。
```

## 6.5 多系统关联功能

同一个知识点可以属于多个系统。

例如：

### Token

- 智能系统：大模型基础单位
- 财富系统：Token使用量对应算力需求和商业成本
- 产业系统：推动GPU、HBM、数据中心需求
- 范式系统：软件从“功能调用”变成“智能调用”

### 高级感

- 美学系统：视觉比例、留白、质感
- 意义系统：身份认同、品牌溢价
- 认知系统：人对秩序和稀缺性的偏好
- 产品系统：提高用户信任和转化

系统需要支持一个知识点挂载多个系统，并展示交叉关联。

## 14. 关键体验要求

1. 用户添加知识要足够快，不能复杂。
2. AI生成内容后，用户可以编辑，而不是强制接受。
3. 每张卡片都必须鼓励用户写“我的启发”。
4. 系统要强调知识之间的连接，而不是孤立笔记。
5. 页面风格要高级、简洁、有秩序感。
6. 知识库不是为了收藏，而是为了输出判断。
7. 所有内容尽量支持Markdown。
8. 支持导入导出，避免用户被平台锁死。
