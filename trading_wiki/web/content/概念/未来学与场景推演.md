# 未来学与场景推演

## 概念定义

未来学不是算命式预测，而是系统识别驱动因素、不确定性和可能场景。它的目标是让人在不确定环境中提前准备、发现机会、降低盲区。

## 场景推演流程

| 步骤 | 问题 | 产出 |
| --- | --- | --- |
| 明确焦点 | 要研究哪个问题的未来 | AI 教育、机器人劳动力、医疗支付 |
| 找驱动因素 | 哪些力量会改变它 | 技术、政策、成本、人口、文化 |
| 找关键不确定性 | 哪些变量最重要且最不确定 | 监管强弱、成本下降速度 |
| 构建场景 | 组合出多种可能未来 | 高增长、低增长、监管收紧、生态开放 |
| 找早期信号 | 哪些迹象说明某场景正在发生 | 订单、政策、用户行为、资本开支 |
| 制定选项 | 每个场景下如何行动 | 投资、学习、产品、组织调整 |

## 四类未来

| 类型 | 含义 |
| --- | --- |
| 可能未来 | 理论上可能发生的多种路径 |
| 可行未来 | 在资源、技术和制度上有机会发生 |
| 可能性较高未来 | 根据当前趋势更容易发生 |
| 偏好未来 | 我们希望主动塑造的方向 |

## 常见工具

| 工具 | 用途 |
| --- | --- |
| PESTEL | 政治、经济、社会、技术、环境、法律分析 |
| 驱动因素地图 | 找出影响未来的力量 |
| 二轴场景 | 用两个关键不确定性生成四象限 |
| 回溯规划 | 从未来目标倒推今天行动 |
| 弱信号扫描 | 捕捉早期变化 |

## 常见误区

| 误区 | 更准确理解 |
| --- | --- |
| 未来学就是预测一个答案 | 重点是准备多种可能 |
| 趋势会线性延续 | 技术、政策和文化会出现拐点 |
| 只看技术趋势 | 还要看制度、商业、社会接受度 |
| 场景越多越好 | 场景应足够少、足够有区分度 |
