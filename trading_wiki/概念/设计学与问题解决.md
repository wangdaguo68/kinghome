# 设计学与问题解决

## 概念定义

设计学研究人如何在约束中创造方案。设计不是单纯美化，而是把功能、形式、材料、用户、商业、技术和文化组织成可用、可感知、可传播的系统。

## 设计的四个约束

| 约束 | 问题 |
| --- | --- |
| 人 | 谁使用？有什么动机、能力和情境？ |
| 功能 | 需要完成什么任务？ |
| 技术 | 能用什么材料、工艺、平台和能力实现？ |
| 意义 | 它传达什么身份、价值和文化位置？ |

## 双钻模型

常见设计流程可以理解为：发散理解问题，收敛定义问题；再发散探索方案，收敛交付方案。

| 阶段 | 目标 | 产出 |
| --- | --- | --- |
| Discover | 探索真实情境 | 访谈、观察、问题清单 |
| Define | 定义关键问题 | 设计目标、用户旅程、约束 |
| Develop | 生成多个方案 | 原型、概念、风格方向 |
| Deliver | 验证并落地 | 可用产品、服务、视觉系统 |

## 好设计的判断

| 标准 | 说明 |
| --- | --- |
| 有用 | 解决真实问题 |
| 可用 | 用户知道怎么用，错误成本低 |
| 可理解 | 信息结构清楚 |
| 可持续 | 维护成本和扩展成本可控 |
| 有意义 | 形式和品牌语境一致 |
| 有审美 | 比例、节奏、材质、色彩和细节统一 |

## 常见误区

| 误区 | 更准确理解 |
| --- | --- |
| 设计就是视觉 | 视觉只是设计的一部分 |
| 用户说什么就做什么 | 用户反馈是线索，不是方案本身 |
| 设计思维万能 | 它适合复杂问题探索，但不能替代专业能力 |
| 极简就是高级 | 极简需要信息结构和细节支撑，否则只是空 |
