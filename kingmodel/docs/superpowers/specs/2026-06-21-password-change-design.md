# 系统设置密码修改功能设计

## 目标

为单用户 KingModel 系统增加站内密码修改功能。功能保持简单，不引入密码复杂度、会话版本或多设备管理。

## 交互设计

“系统设置”工作区展示“账户安全”卡片，包含三个密码输入框：

- 当前密码
- 新密码
- 确认新密码

提交前仅检查字段非空以及两次新密码一致。提交成功后清空表单并展示成功提示，当前登录会话继续有效。新密码立即可用于登录，旧密码立即失效。

## 后端设计

新增已认证接口 `POST /api/auth/password`，请求体包含 `current_password` 和 `new_password`。

处理流程：

1. 从当前签名 Cookie 获取用户名。
2. 使用 Argon2 验证当前密码。
3. 拒绝空密码。
4. 使用 Argon2 生成新哈希并更新 SQLite 用户记录。
5. 返回成功结果，不清除或重签当前 Cookie。

接口不修改服务器 `.env`。数据库中的密码哈希是运行时认证的唯一有效值。

## 错误处理

- 当前密码错误：返回 `400` 和明确提示。
- 字段为空：返回 `422`。
- 未登录：返回 `401`。
- 前端显示接口返回的错误，不清空用户已经输入的内容。

## 测试与验收

- 当前密码错误时数据库不变。
- 正确修改后旧密码登录失败，新密码登录成功。
- 修改成功后原 Cookie 仍可访问已认证接口。
- 前端类型检查与生产构建通过。
- 部署后执行上述登录和会话验证。

## 非目标

- 密码长度或字符复杂度规则
- 找回密码
- 多用户管理
- 注销其他设备
- 密码历史记录
