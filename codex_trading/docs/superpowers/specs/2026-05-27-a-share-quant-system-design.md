# A 股短线量化交易系统设计

日期：2026-05-27

## 目标

把 `data/交易框架.md`、日/周/月复盘框架中的人工交易体系，转成一个可回测、可复盘、可扩展到模拟交易和自动下单的本地量化系统。

首期目标不是直接实盘下单，而是先完成：

- 免费行情数据接入和本地缓存
- 情绪周期 `CycleTag` 计算
- 策略规则引擎
- 回测和卖出后回溯
- 日/周/月复盘报告
- 高保真 Web 工作台

实盘自动下单放到后续阶段，必须在回测、模拟盘、风控闸门稳定后接入。

## 市场与数据

首期聚焦 A 股短线体系。

数据源优先使用免费接口，如 AkShare/Tushare 的公开能力。数据层必须做成适配器，未来替换为付费行情源时，不影响策略、回测和 UI。

免费数据无法完整还原竞价撤单、封单变化、盘口扫单和真实打板成交，因此首期回测定义为“规则验证与近似回测”，不承诺盘口级真实成交。

## 核心模块

### 1. 数据中心

职责：

- 拉取 A 股日线、分钟线、指数、涨跌停、成交额、板块相关数据
- 计算或导入 `880005` 红盘家数
- 缓存到本地数据库
- 对外提供统一查询接口

首期数据库使用 SQLite，后续可切换 DuckDB/PostgreSQL。

### 2. 情绪周期引擎

根据文档规则计算：

- `RedCount`
- `MA3`
- `MA5`
- MA3/MA5 趋势
- `CycleTag`

周期标签包括：

- `IcePoint`
- `TurnUp`
- `MainRally`
- `Climax`
- `TurnDown`
- `Downtrend`
- `LowShake`
- `HighShake`

### 3. 策略规则引擎

首期优先实现较容易用免费数据近似验证的三类模式：

- `ExtremeArbitrage`
- `FirstLimit`
- `OneToTwo`

第二期再实现更依赖分时和盘口的模式：

- `SecondWaveWrap`
- `DipBuy`
- `AccumulationIntraday`

每个策略必须声明：

- 启用周期
- 禁用周期
- 入场条件
- 出场条件
- 仓位规则
- 止损规则
- 数据依赖
- 免费数据下的近似假设

### 4. 风控闸门

所有买入信号必须经过统一风控检查：

- 当前 `CycleTag` 是否允许该模式
- 成交额/容量是否达标
- 仓位是否超过周期上限
- 是否设置硬止损
- 是否连续亏损 2 笔后仍买
- 是否割肉当天买新票
- 是否下午指数回落时买入
- 是否属于模式外交易

风控闸门在回测、模拟盘、未来实盘中共用，不能绕过。

### 5. 回测引擎

职责：

- 按日期推进历史行情
- 生成信号
- 执行风控检查
- 模拟成交和持仓
- 触发止损、止盈、时间止损、周期卖出
- 输出交易明细和统计指标

核心指标：

- 总收益
- 年化收益
- 最大回撤
- 胜率
- 盈亏比
- 平均持仓天数
- 模式内/模式外占比
- `CycleTag` 正确率
- 卖出后 3 日涨跌幅回溯

### 6. 复盘系统

根据 `data` 下的日/周/月复盘框架，自动生成 Markdown 或 HTML 报告。

首期报告包括：

- 每日 CycleTag 和可用模式
- 当日信号和风控拒绝原因
- 回测交易明细
- 周度模式表现
- 月度参数校准建议
- 模式外交易归因
- 卖出质量评估

### 7. Web 端

采用“主工作台 + 研究复盘 + 风控执行”的组合设计：

- 主工作台采用黑箱研究终端风：高密度、暗色、指标优先
- 研究复盘采用西蒙斯式研究账本风：证据链、模型纪律、参数校准
- 下单/模拟盘采用风控执行驾驶舱：闸门清晰、人工确认、状态可追踪

首期页面：

- 行情数据
- 周期雷达
- 策略实验
- 回测报告
- 风控执行

## 测试策略

必须覆盖：

- `CycleTag` 计算边界
- 策略启用/禁用周期
- 风控闸门拒绝原因
- 单笔硬止损
- 回测持仓和交易记录
- 卖出后 3 日回溯
- API 基础可用性

开发完成后执行：

- 后端单元测试
- 前端构建检查
- 关键页面手动检查
- 代码审查专家审查

## 代码审查要求

开发完成后调用代码审查专家或子代理审查。

审查重点：

- 策略规则是否和文档一致
- 回测是否存在未来函数
- 风控闸门是否可绕过
- 数据适配器边界是否清晰
- UI 是否只展示模拟/回测，不误导为真实下单
- 测试是否覆盖核心交易风险

## 阶段计划

### 第一阶段：MVP

- 工程搭建
- Demo 数据
- CycleTag 引擎
- 三个基础策略
- 风控闸门
- 回测 API
- 高保真 Web 首页和回测页
- 自动化测试

### 第二阶段：免费数据接入

- AkShare/Tushare 适配器
- 本地数据缓存
- 批量拉取任务
- 更多回测样本

### 第三阶段：模拟盘

- 盘中信号刷新
- 模拟订单
- 人工确认
- 执行日志

### 第四阶段：实盘

- 接入付费行情
- 接入券商交易接口
- 只允许经过风控闸门的订单
- 默认人工确认，稳定后再开放自动确认

## 风险与约束

- 免费数据不适合盘口级打板回测
- A 股自动下单需要券商接口、合规和账户权限
- 实盘前必须完成模拟盘长期验证
- 系统输出不是投资建议，只执行用户定义的交易规则
