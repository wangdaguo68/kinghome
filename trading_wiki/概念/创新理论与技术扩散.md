# 创新理论与技术扩散

## 概念定义

创新理论研究新技术、新产品、新组织方式如何被发明、采用、扩散并改变系统。技术扩散不是只看技术先进性，还要看成本、场景、渠道、标准、生态和用户习惯。

## 扩散曲线

| 群体 | 特征 | 关键问题 |
| --- | --- | --- |
| 创新者 | 愿意尝试新东西 | 技术是否可用 |
| 早期采用者 | 看重领先优势 | 是否形成标杆案例 |
| 早期大众 | 需要可靠和性价比 | 是否跨越鸿沟 |
| 后期大众 | 需要成熟和低风险 | 是否成为标准配置 |
| 落后采用者 | 被环境推动 | 是否已无法避免 |

## 技术产业化条件

| 条件 | 问题 |
| --- | --- |
| 性能 | 是否比旧方案好很多 |
| 成本 | 是否下降到可规模化 |
| 场景 | 是否解决真实高频痛点 |
| 基础设施 | 数据、能源、网络、供应链是否成熟 |
| 标准 | 是否有接口、规范和生态 |
| 组织 | 谁来采购、部署、维护 |
| 监管 | 是否被允许和信任 |

## 判断扩散阶段

| 信号 | 含义 |
| --- | --- |
| Demo 多、订单少 | 仍在概念验证 |
| 标杆客户出现 | 开始有可信应用 |
| 成本快速下降 | 扩散条件改善 |
| 生态伙伴增加 | 标准和互补品成熟 |
| 从预算外变成预算内 | 进入组织常规采购 |
| 从“新奇”变成“默认” | 扩散基本完成 |
