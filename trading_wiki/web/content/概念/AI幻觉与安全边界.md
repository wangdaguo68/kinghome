# AI幻觉与安全边界

## 一句话解释

AI 幻觉是模型生成看似合理但不真实或无依据内容的现象。它来自生成式模型的预测机制、训练数据局限、检索错误和缺乏真实世界校验。

## 常见类型

| 类型 | 表现 |
| --- | --- |
| 事实幻觉 | 编造人物、论文、公司、数据 |
| 引用幻觉 | 给出不存在或不支持结论的来源 |
| 推理幻觉 | 中间步骤看似合理但逻辑错误 |
| 工具幻觉 | 误读工具输出或假装调用成功 |
| 过度自信 | 置信语气高于证据强度 |

## 降低方法

- 要求引用和证据。
- 对关键事实做外部验证。
- 使用 RAG，但要验证检索片段。
- 对高风险场景加入人工审核。
- 让模型输出“不确定”和反证。

## 高风险场景

医疗、法律、金融交易、公司公告、药品剂量、身份认证、自动执行命令，都不能只靠模型输出。

## 我的启发

AI 是认知放大器，不是事实保证器。越重要的判断，越要把 AI 当成生成假设的助手，而不是最终裁判。
