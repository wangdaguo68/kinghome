# 规则底座与多任务机器学习交易计划引擎设计

## 1. 目标

系统每天收盘后生成下一交易日计划，最多输出综合确定性最高的 3 只股票，也允许少于 3 只或空仓。用户从计划中选择 1 至 3 只执行。

系统以稳定、可解释和风险可控为第一目标。机器学习用于概率估计、排序、周期选择和规则阈值校准，不绕过风险硬规则，也不承诺确定盈利或固定复利。

## 2. 设计原则

- 规则决定市场是否允许交易以及候选是否合规。
- 模型估计条件概率、预期收益、回撤和可成交性。
- 市场、板块和个股采用分层模型，不使用单一端到端黑箱。
- 训练和实盘使用同一套特征生成代码。
- 所有特征必须在决策时间点真实可得，禁止未来数据泄漏。
- 每个预测记录数据版本、特征版本、模型版本和解释证据。
- 数据或模型不可信时允许空仓，并回退到规则模式。
- 模型必要数据使用免费行情和本地历史，不依赖高频通达信 MCP。

## 3. 总体决策链路

```text
数据完整性检查
  → 风险硬门槛
  → 赚钱/亏钱效应
  → 市场风格概率
  → 周期阶段概率
  → 板块迁移与主线概率
  → 板块内核心候选
  → 分类型收益/回撤模型
  → 风险调整排序
  → Top1–Top3 或空仓
```

## 4. 系统边界

### 4.1 交易范围

- 可交易：沪深主板、创业板。
- 观察但不可进入计划：科创板、北交所。
- 排除：ST、退市风险、上市时间过短、流动性不足、停牌及明显不可成交标的。

### 4.2 决策时点

- 15:10 后生成下一交易日计划。
- 当日计划发布后锁定。
- 盘中页面只读取计划，不重复推理、不改变收盘模型结果。
- 盘中条件只负责确认或否定收盘预案。

## 5. 特征存储

新增不可修改的每日特征快照。每条记录包含 `as_of_trade_date`、采集时间、数据来源、数据完整性和特征版本。

训练、回测和线上推理必须调用相同的特征接口。修正历史数据时创建新版本，不静默覆盖旧版本。

## 6. 市场层特征

### 6.1 赚钱效应

- 上涨家数比例和全市场涨跌幅中位数。
- 涨停数量、真实连板数量、最高板和晋级率。
- 昨日涨停、连板及强势股的次日开盘和收盘溢价。
- 炸板修复率和创 20 日新高后的延续率。
- 成交额前 100 的收益中位数、创新高比例和趋势延续率。
- 创业板 20cm 涨停数量及次日溢价。
- 最近 1、3、5 日赚钱效应及变化斜率。

### 6.2 亏钱效应

- 下跌家数比例、收益中位数和跌停数量。
- 炸板率、炸板股收盘亏损和次日亏损。
- 昨日涨停、连板、高位股的次日负溢价。
- 天地板、准天地板和高位 A 杀数量。
- 高成交额核心的破位比例及收益中位数。
- 负反馈是否从高位向中低位、从题材向容量扩散。
- 最近 1、3、5 日亏钱效应及变化斜率。

### 6.3 风格特征

- 大市值与小市值分组收益、成交占比和创新高比例。
- 趋势核心组合与连板投机组合的收益、波动和持续性。
- 市场成交额向权重容量或小票题材的集中方向。
- 黄白线差值及持续时间仅作辅助；缺失时模型必须可运行。
- 趋势和投机允许同时为高概率，也允许同时失效。

## 7. 板块层特征

- 1、3、5、10 日涨幅和相对全市场超额收益。
- 成交额、市场成交占比和变化速度。
- 板块上涨家数比例与收益中位数。
- 涨停、连板、20cm、创新高和趋势上涨数量。
- 先锋、情绪龙头、趋势中军和低位补涨的梯队完整性。
- 昨日强势股次日溢价、分歧修复速度和回流强度。
- 最近 3 日强势板块重合度与排名迁移。

板块迁移标签包括：板块内高低切、同级板块横切、板块外切、双主线和旧主线退潮。

## 8. 个股层特征

- 真实连板高度、近 5 日涨停次数、封板时间和炸板次数。
- 1、3、5、10 日收益、波动率和最大回撤。
- 成交额、换手率、流通市值和相对历史分位。
- 相对板块和全市场的主动性。
- 是否为板块最早涨停、最高板、最大容量、20cm 先锋。
- 板块强度、主线概率、风格概率和周期概率。
- 一字板、停牌、涨跌停及其他可成交性特征。

## 9. 多任务模型

首选 LightGBM 或 CatBoost 等表格树模型，原因是适合中等规模结构化数据、可处理非线性和特征交互，并可输出特征贡献。第一阶段不使用强化学习。

### 9.1 市场风格模型

输出：

- `P(trend)`：趋势风格概率。
- `P(speculation)`：投机连板风格概率。
- `P(mixed)`：两种风格共振概率。
- `P(no_edge)`：两种模式均缺乏优势的概率。

标签通过比较未来趋势核心组合与投机组合的可执行收益和风险生成，不由人工主观标注。

### 9.2 周期阶段模型

输出主升、高位震荡、混沌轮动、退潮、试错修复的概率分布。输入包含赚钱效应、亏钱效应、主线集中度、持续性及其 3 至 5 日轨迹。

周期输出必须包含主状态、次状态、概率和触发状态切换的证据。

### 9.3 板块模型

分别预测板块未来 1 日和 3 日超额收益、延续概率和回撤概率。主线确认同时要求：

- 概率达到校准阈值。
- 至少连续两个交易日有持续性证据，首日新题材除外。
- 成交承载和梯队结构通过硬过滤。
- 明显领先第二名或被标记为双主线。

### 9.4 个股模型

不同交易模式使用独立模型：

- 连板模型：次日可成交概率、开盘至收盘收益和大幅回撤概率。
- 趋势模型：未来 3、5、10 日收益和最大回撤。
- 20cm 模型：未来 1、3 日溢价和回撤风险。

个股模型不得跨类型直接比较原始概率，必须经过概率校准和风险调整后统一排序。

## 10. 标签定义

- 所有计划以当日收盘后生成，标签从下一交易日开始计算。
- 日频第一版使用次日开盘价作为可执行基准。
- 一字涨停、停牌或开盘不可成交的样本标记为不可成交，不按理论收益计入成功。
- 连板标签：次日开盘至收盘收益、次日最大回撤和可成交性。
- 趋势标签：次日开盘至未来 3、5、10 日收盘收益及区间最大回撤。
- 20cm 标签：次日和未来 3 日收益及最大回撤。
- 板块标签：未来 1、3 日相对全市场超额收益。

未来具备分钟数据后，可将入场基准升级为 9:35–10:00 可成交均价，但必须新建标签版本。

## 11. 风格与交易周期

交易周期由市场风格、周期阶段和个股类型共同决定：

- 投机风格占优且连板核心通过：隔日模式。
- 趋势风格占优且趋势核心通过：比较 3、5、10 日模型，选择风险调整收益最高周期。
- 20cm 弹性核心：比较 1、3 日模型。
- 趋势与投机共振：允许两类计划并存，但分别使用对应周期。
- 高位震荡或轮动：缩短周期并提高概率门槛。
- 退潮：原则上空仓；只有修复概率和可成交性同时通过才允许少量计划。

## 12. Top3 选择

候选必须先通过：

- 市场许可。
- 主线或高概率新主线。
- 数据完整性。
- 可成交性。
- 预期收益覆盖手续费和滑点。
- 回撤风险不超过当前周期门槛。
- 模型未发生漂移。

统一排序分：

```text
综合分 = 校准上涨概率 × 预期收益
       − 最大回撤风险
       − 不可成交风险
       − 风格错配惩罚
       − 数据不确定性惩罚
```

综合结果归一化为 0–100 分的“交易计划综合得分”，分项构成为：

- 校准上涨概率：30 分。
- 风险调整后的预期收益：25 分。
- 主线地位与核心辨识度：15 分。
- 市场风格与周期匹配：10 分。
- 可成交性与流动性：10 分。
- 数据完整性与模型可靠性：10 分。

最大回撤、不可成交、风格错配和数据不确定性先形成扣分，再将最终结果限制在 0–100 分。页面同时展示六个分项及主要扣分原因，不能只显示一个无法解释的总分。

最多输出 3 只，不足不补。Top1、Top2、Top3 必须按综合得分从高到低排列；同分时依次比较回撤风险、上涨概率、可成交性和代码，保证排序稳定。默认同一板块最多 2 只；市场存在唯一确认主线时允许 3 只同板块。

综合得分只用于同一交易日候选的横向比较，不代表收益率、胜率或确定盈利。入选门槛由走步回测校准，未达到门槛时允许少于 3 只或空仓。

没有标的达标时输出“今日无交易计划”，并列出被市场许可、主线、概率、回撤或可成交性过滤的数量。

## 13. 每只计划的输出

- Top1–Top3 排名。
- 0–100 分综合得分及六项分项得分。
- 主要加分依据和风险扣分原因。
- 股票、代码、板块和核心身份。
- 市场风格、周期阶段和主线概率。
- 上涨概率、预期收益和回撤概率。
- 建议观察周期：隔日、1 日、3 日、5 日或 10 日。
- 次日观察与触发条件。
- 失效条件。
- 数据来源和置信度。
- 模型版本、特征版本及历史同类样本表现。

计划是条件预案，不输出无条件买入指令。

## 14. 训练与验证

### 14.1 数据范围

- 首次回补至少 3 年历史，目标 5 年。
- 每日自动追加市场、板块、个股特征及未来标签。
- 训练标签来自行情结果，不使用用户最终选择作为标签，避免选择偏差。

### 14.2 时间走步验证

- 禁止随机切分。
- 训练、验证、测试严格按时间前后排列。
- 相邻集合设置至少等于最大预测周期的 10 个交易日隔离区。
- 回测加入手续费、滑点、涨跌停、停牌和不可成交约束。
- 不同市场风格、周期和个股类型分别评估。

### 14.3 评价指标

- Top1 和 Top3 收益。
- 达标计划胜率与盈亏比。
- 最大回撤和连续亏损次数。
- 风险调整收益和换手率。
- 概率校准误差。
- 不同时间窗口与市场状态下的稳定性。
- 空仓过滤对回撤和收益的贡献。

普通分类准确率不得作为主要升级依据。

## 15. Champion/Challenger

- 线上模型为 Champion。
- 每周训练 Challenger，但不自动替换。
- Challenger 必须在多个走步窗口中同时改善收益、回撤和概率校准。
- 最近单一窗口表现更好不能触发升级。
- 每次升级保留旧模型和完整评估报告。
- 数据漂移、概率失准或连续异常亏损触发回滚规则。

模型样本不足时，规则系统主导；模型仅显示辅助概率且不得扩大仓位。

## 16. 系统模块

- `feature_store`：每日特征快照和版本管理。
- `market_regime`：赚钱/亏钱效应、风格和周期概率。
- `sector_ranker`：板块迁移、主线概率和延续预测。
- `stock_ranker`：连板、趋势和 20cm 模型。
- `portfolio_selector`：规则过滤、风险约束和 Top3。
- `model_registry`：Champion、Challenger、升级和回滚。
- `outcome_tracker`：回填 1、3、5、10 日结果。
- `training_pipeline`：训练、走步回测和报告。

模块通过版本化数据结构通信，不直接读取彼此内部状态。

## 17. 每日运行

1. 15:10 获取免费收盘数据并固化快照。
2. 计算市场、板块和个股特征。
3. 规则引擎执行风险硬过滤。
4. 模型输出风格、周期、板块和个股概率。
5. 组合选择器生成 Top1–Top3 或空仓计划。
6. 原子发布并锁定当日计划。
7. 后续交易日自动回填结果。
8. 每周运行 Challenger 训练与评估。

盘中页面读取已发布计划，不重复模型推理或调用通达信。

## 18. 界面

计划页展示：

- 市场许可。
- 风格概率和周期概率。
- 主线及延续概率。
- Top1–Top3 计划卡片。
- 触发、失效、周期、概率、预期收益和回撤。
- 无计划时的过滤原因。
- 模型健康、数据完整性和版本。

复盘页展示计划后续 1、3、5、10 日结果、未执行计划结果和模型校准情况。

## 19. 故障与降级

- 模型不可用：回退规则计划并明确标记。
- 特征不完整：不生成新计划。
- 免费数据失败：保留上一交易日页面用于复盘，但标记不可作为当日新开仓计划。
- 模型漂移：停止机器学习计划并回滚 Champion 或规则模式。
- 通达信仅用于每日最多 5 只原因补查，不是模型必要依赖。

## 20. 验收标准

- 交易周期确实由风格、阶段和个股类型共同决定。
- 赚钱、亏钱、风格、周期、主线和核心均有明确特征及概率解释。
- 每日最多 3 只，允许少于 3 只和空仓。
- Top3 按 0–100 综合得分严格降序，并可查看分项和扣分依据。
- 训练与线上特征逐字段一致。
- 时间走步测试无未来数据泄漏。
- 回测包含真实交易限制和成本。
- 模型升级可审计、可回滚。
- 盘中读取计划不消耗通达信 Token。

## 21. 分阶段实施

该目标拆为四个可独立验收的子项目，禁止一次性上线未经验证的完整模型：

1. 特征与规则基线：完成历史数据、特征快照、真实标签和完整规则评分。
2. 市场与板块模型：上线风格、周期、板块迁移和主线概率，但暂不自动选择个股。
3. 个股与 Top3 模型：上线三类个股模型、周期选择、风险调整排序和空仓机制。
4. 持续学习系统：上线结果跟踪、Champion/Challenger、漂移监控、回滚和模型复盘界面。

每个阶段必须通过独立走步回测和线上影子运行，再进入下一阶段。
